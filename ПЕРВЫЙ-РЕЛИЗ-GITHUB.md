# Первый релиз на GitHub (Murs2024 / youtube-dl-gui-master)

Пошаговая инструкция: что именно заполнить на странице **New release** и откуда берётся ссылка на скачивание — для пользователей и для встроенной проверки обновлений в **Murs Media**.

---

## Что подготовить до захода на GitHub

### Вариант А — один скрипт: полный комплект в `release-staging`

В корне репозитория выполните:

```powershell
powershell -ExecutionPolicy Bypass -File .\prepare-github-release.ps1
```

По умолчанию тег папки/ZIP — **`1.0.0`**. Свой тег:

```powershell
powershell -ExecutionPolicy Bypass -File .\prepare-github-release.ps1 -VersionTag 1.0.1
```

Уже собрали Release вручную:

```powershell
powershell -ExecutionPolicy Bypass -File .\prepare-github-release.ps1 -SkipBuild
```

Скрипт:

1. запускает **`build.ps1`** (если не указан `-SkipBuild`);
2. копирует **всё** из `youtube-dl-gui\bin\Release\` в папку  
   **`release-staging\MursMedia-<тег>-portable\`** (эта папка в **`.gitignore`**, в Git не попадает);
3. создаёт архив **`release-staging\MursMedia-<тег>-portable.zip`** — удобно выложить вторым файлом на релиз;
4. пишет **`release-staging\MursMedia-<тег>-portable-exe-sha256.txt`** со строкой **`exe sha256: ...`** для копирования в описание релиза;
5. из корневого **`github-release-body-template.md`** (плейсхолдеры `{{TAG}}`, `{{SHA256}}`) генерирует **`release-staging\MursMedia-<тег>-portable-GITHUB-RELEASE-BODY.md`** — **его целиком вставляйте** в описание релиза на GitHub (отдельная копия шаблона в `release-staging` не создаётся).

Папка `release-staging` создаётся **локально** рядом с проектом; бинарники в репозиторий не коммитятся.

### Пример: что лежит внутри после скрипта (типичное дерево)

Корень: `release-staging\MursMedia-1.0.0-portable\` (имя зависит от `-VersionTag`).

```text
MursMedia-1.0.0-portable\
├── youtube-dl-gui.exe          ← главный исполняемый файл (имя не менять для автообновления)
├── youtube-dl-gui.ini          ← может появиться после запуска программы у вас локально; в чистой сборке часто нет
└── lang\
    ├── Arabic.ini
    ├── Brazilian_Portuguese.ini
    ├── Dutch.ini
    ├── English.ini
    ├── French.ini
    ├── German.ini
    ├── Italian.ini
    ├── Japanese.ini
    ├── Russian.ini
    ├── Simplified Chinese.ini
    └── Spanish.ini
```

Отдельный проект **youtube-dl-gui-updater** кладёт `youtube-dl-gui-updater.exe` в свою `bin\Release`; основная программа при обновлении подставляет updater из ресурсов рядом с exe — для пользователя достаточно папки с **`youtube-dl-gui.exe`** и **`lang\`**.

**yt-dlp** и **ffmpeg** в эту папку сборка не копирует: их пользователь указывает в настройках или кладёт рядом сам (как в основном README).

### Вариант Б — вручную

1. Соберите **Release**: `build.ps1`.
2. Берите файлы из **`youtube-dl-gui\bin\Release\`** — как минимум **`youtube-dl-gui.exe`** и папку **`lang\`** (если отдаёте «портатив» целиком).
3. SHA-256 для описания релиза:

   ```powershell
   Get-FileHash -Algorithm SHA256 .\youtube-dl-gui.exe
   ```

   (выполните в папке, где лежит `youtube-dl-gui.exe`).

### Пример: как заполнить первый релиз на GitHub (готовый текст)

Ниже — **образец**, который можно почти целиком скопировать; подставьте **свой** хеш из `*-exe-sha256.txt` или из `Get-FileHash`.

| Поле на сайте | Что вписать (пример) |
|----------------|----------------------|
| **Choose a tag** | `1.0.0` (без префикса `v`) |
| **Target** | `main` |
| **Release title** | `Murs Media 1.0.0 — первый релиз` |

**Описание (Release notes) — пример:**

```markdown
## Murs Media (GUI для загрузки и конвертации)

**Автор:** Ольга Шевелева  
**Дата сборки / релиза:** 08.04.2026  

### Состав
- Скачайте **youtube-dl-gui.exe** для обновления из приложения или одиночного файла.
- Либо архив **MursMedia-1.0.0-portable.zip** — распакуйте в любую папку, запускайте `youtube-dl-gui.exe`.

### Зависимости
Установите [yt-dlp](https://github.com/yt-dlp/yt-dlp/releases) и при необходимости **ffmpeg**; пути можно указать в настройках программы.

exe sha256: 0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef
```

Строка **`exe sha256: ...`** должна быть **реальной** — замените пример на вывод из скрипта или `Get-FileHash` (64 символа, латиница a–f и цифры).

**Assets (файлы к релизу) — пример:**

1. **`youtube-dl-gui.exe`** — из `release-staging\MursMedia-1.0.0-portable\` (или из `youtube-dl-gui\bin\Release\`). Имя на GitHub: **строго** `youtube-dl-gui.exe`.
2. **`MursMedia-1.0.0-portable.zip`** — из `release-staging\` (необязательно, но удобно для «скачал и распаковал»).

После **Publish release** прямая ссылка на exe будет:

`https://github.com/Murs2024/youtube-dl-gui-master/releases/download/1.0.0/youtube-dl-gui.exe`

(тег в URL должен совпадать с созданным тегом **буква в букву**).

## Шаг 1. Открыть создание релиза

1. Зайдите в репозиторий: [https://github.com/Murs2024/youtube-dl-gui-master](https://github.com/Murs2024/youtube-dl-gui-master).
2. Справа найдите блок **Releases** → **Create a new release** (или через **Releases** → кнопка **Draft a new release**).

---

## Шаг 2. Поле «Tag» (тег версии)

1. Нажмите **Select tag** → **Create new tag**.
2. Введите тег **без буквы `v` в начале** — так устроен разбор версии в этой программе.

   **Примеры подходящих тегов:**

   - `1.0.0` — первый стабильный релиз;
   - `1.0.1`, `1.1.0` — следующие релизы;
   - `1.0.1-1` — пре-релиз / бета (суффикс через дефис; должен совпадать с логикой `Program.CurrentVersion` в коде).

   **Не подойдёт** для логики версий в приложении: `v1.0.0` (с буквой `v`).

3. **Target**: оставьте ветку **`main`** (или ту, с которой вы реально собираете релиз).

---

## Шаг 3. Поле «Release title» (заголовок)

Это только заголовок для людей на странице релизов. На работу автообновления он не влияет.

**Пример:**

```text
Murs Media — первый публичный релиз (GUI)
```

или короче:

```text
Murs Media 1.0.0
```

---

## Шаг 4. Описание релиза (Release notes)

В большом поле **Describe this release** можно:

- нажать **Generate release notes** — подтянутся заголовки коммитов (при желании отредактируйте);
- дописать вручную важное для пользователей.

**Имеет смысл явно указать:**

- название продукта (**Murs Media**);
- автора (**Ольга Шевелева**);
- дату сборки или релиза;
- кратко: что умеет, откуда брать **yt-dlp** / **ffmpeg**, если не кладёте их в архив.

**Обязательно для корректной проверки exe при обновлении** добавьте отдельной строкой (подставьте свой хеш):

```text
exe sha256: <вставьте_64_символа_из_Get-FileHash>
```

Формулировка именно с **`exe sha256:`** важна — так ищет строку код обновления.

---

## Шаг 5. Прикрепить файл программы (Assets)

1. В зоне **Paste, drop, or click to add files** перетащите **`youtube-dl-gui.exe`** из `youtube-dl-gui\bin\Release\`.
2. После загрузки GitHub покажет файл в списке вложений.

**Критично:** имя файла в релизе должно быть **`youtube-dl-gui.exe`** (именно так).  
Встроенное обновление собирает прямую ссылку вида:

`https://github.com/Murs2024/youtube-dl-gui-master/releases/download/<тег>/youtube-dl-gui.exe`

Если вы переименуете файл (например, в `MursMedia.exe`), по этой ссылке скачивание **сломается**, пока вы не измените логику в коде.

**Про ZIP:** можно дополнительно выложить архив со всеми DLL и зависимостями для ручной установки — это удобно людям. Но для кнопки «Обновить» в программе по-прежнему нужен отдельный ассет **`youtube-dl-gui.exe`** с нужным именем (или доработка кода под другое имя).

---

## Шаг 6. Пререлиз и «Latest»

- Если отметить **Set as a pre-release**, релиз не будет считаться «последним стабильным» на GitHub, пока вы не снимете эту отметку.
- Обычный опубликованный релиз без пререлиза становится **Latest** — его и использует GitHub по умолчанию и API **`.../releases/latest`**, если в настройках приложения не включены бета-релизы.

Опубликуйте релиз кнопкой **Publish release** (или сохраните черновик **Save draft**, доработайте и опубликуйте позже).

## Откуда берётся «ссылка на релиз», чтобы скачивалось

### Для людей (страница релизов)

После публикации постоянная страница всех релизов:

[https://github.com/Murs2024/youtube-dl-gui-master/releases](https://github.com/Murs2024/youtube-dl-gui-master/releases)

На ней у каждого релиза есть блок **Assets** — там прямые ссылки на каждый загруженный файл (в том числе на **`youtube-dl-gui.exe`**). Можно скопировать ссылку через правый клик → **Copy link**.

### Предсказуемая прямая ссылка на exe (удобно для скриптов и проверки)

Формат:

```text
https://github.com/Murs2024/youtube-dl-gui-master/releases/download/<ТЕГ>/youtube-dl-gui.exe
```

**Пример:** если тег **`1.0.0`**, то:

```text
https://github.com/Murs2024/youtube-dl-gui-master/releases/download/1.0.0/youtube-dl-gui.exe
```

Откройте эту ссылку в браузере — должен начаться скачивание файла. Если 404, проверьте: тег написан **точно так же**, как в релизе, и файл в релизе называется **`youtube-dl-gui.exe`**.

### Для программы Murs Media («Проверить обновления»)

Отдельно в настройках приложения **URL релиза вводить не нужно**. Программа уже обращается к API GitHub для репозитория **`Murs2024/youtube-dl-gui-master`** и сама:

- узнаёт последний релиз (или список пререлизов — в зависимости от настроек);
- сравнивает версию из **тега** с версией запущенного exe;
- при обновлении качает файл по тому же шаблону прямой ссылки, что выше.

Достаточно один раз правильно опубликовать релиз с **корректным тегом** и ассетом **`youtube-dl-gui.exe`**, затем пересобрать и распространять exe, в котором зашита актуальная логика обновлений (как в вашей текущей ветке кода).
